⏺ Now I have a clear picture. Here's the core tension and the clean solution:

  The tension: Phase5 removes unfold from the DSL by making root_type=fn — the
  evaluator implicitly wraps each enumerated fn as unfold(x[0], T, fn_val). But
  phase2 false-belief tasks need to search over grid transformations (phantom
  wall placement), which currently requires unfold to be an explicit DSL node so
   coordinate enumeration can happen inside the program tree.

  The resolution — separate belief synthesis from behavior synthesis:

  The synthesized program should never be a trajectory. It should be one of:

  1. root_type=fn — enumerate a transition function grid -> grid. Evaluator:
  unfold(actual_g, T, fn_val). Already done in phase5.
  2. root_type=grid (new) — enumerate the agent's believed initial world state
  (a grid value). Evaluator: unfold_belief(actual_g, believed_g, step_fn) where
  step_fn is either hardcoded or provided from prior behavior synthesis.

  For phase2 false-belief tasks with root_type=grid, the DSL needs only:
  place_wall, ig_i terminals, and int terminals. The synthesized program is
  literally place_wall(ig_i, r, c) — the agent's believed starting world.
  unfold, mask, and approach are all implicit in the evaluator (and mask becomes
   redundant entirely, since unfold_belief already operates in the actual world
  — the phantom wall is never in the output trajectory).

  For nav tasks (no false belief), the believed world = actual world, so the
  program is just ig_i.

  Stitch then discovers fn_belief($ig, $r, $c) = place_wall($ig, $r, $c) and
  fn_navigate($ig) = $ig — the belief function vs. the pass-through. That's the
  structural signature you want.

  For phase6, this scales directly: root_type=agent_to_grid where programs have
  type int -> grid (maps agent id to their believed world). The synthesizer
  finds fn_assigns_worlds($agent) = if_fn($agent == 1, place_wall($ig, r1, c1), 
  place_wall($ig, r2, c2)) or whatever structure emerges. This is the belief
  function in the full sense.

  Implementation sketch for phase2 refactored:

          try:

          except Exception:

  The ig_map now includes a step_fn (hardcoded approach(1,2) for phase2, or
  synthesized from a prior behavior-search run).

  Summary: Two separate enumeration modes — behavior search (root_type=fn) and
  belief search (root_type=grid) — with unfold always implicit in the evaluator.
   The synthesized objects are meaningful: behaviors are transition functions,
  beliefs are believed world states. Phase6 just generalizes root_type=grid to
  root_type=agent_to_grid.

⏺ Here's an overview of each phase:

  ---
  phase1.py — structural abstraction baseline

  Navigation on a 4×4 grid with fixed walls. Tasks are trivial to solve: the
  program is always unfold(ig_i, T, approach(1,2)). The point isn't the solution
   — it's showing that stitch can compress 8 such programs into a reusable nav
  primitive: fn_nav($grid, $T) = (unfold $grid $T navigate). The walls are baked
   into ig_i so stitch doesn't need place_wall at all.

  ---
  phase2.py — false belief, world-level description

  Mix of nav and false-belief tasks. False-belief tasks show an agent taking a
  "suboptimal" path that only makes sense if there's a phantom wall the agent
  believes in. DSL contains explicit unfold, place_wall, and mask. Solution
  structure: mask(unfold(place_wall(ig_i, r, c), approach(1,2)), 3) — navigate
  on the modified world, hide the phantom wall from output. Stitch discovers
  fn_belief($grid, $r, $c) with the mask-navigate-under-hidden-wall pattern. The
   mask is the semantic signature of belief at the world level.

  ---
  phase2b.py — false belief, mentalistic description

  Same tasks as phase2 but the DSL replaces mask + place_wall with two explicit
  mental-state primitives: add_phantom_wall (constructs the agent's false model)
   and unfold_belief (navigates on the believed grid, replays moves on the
  actual world). Solution: unfold_belief(ig_i, add_phantom_wall(ig_i, r, c), 
  approach(1,2)). ig_i appears twice — once as the actual world, once as the
  anchor for the belief. Stitch discovers fn_belief($ig, $r, $c) where the
  shared $ig is the structural signature that the agent's belief is grounded in
  the actual world.

  ---
  phase2c.py (new) — false belief, belief search only

  Same tasks, but unfold is completely absent from the DSL. Instead of
  enumerating trajectory-producing programs, the synthesizer enumerates
  grid-typed programs (believed initial worlds) and the evaluator implicitly
  wraps them: unfold_belief_steps(actual_g, believed_g, T, approach(1,2)). Nav
  tasks solved by ig_i (1 node — believed world = actual world). False-belief
  tasks solved by place_wall(ig_i, r, c) (4 nodes). mask also disappears —
  unfold_belief_steps naturally returns the actual-world trajectory. Stitch
  discovers fn_belief($ig, $r, $c) = (place_wall $ig $r $c).

  ---
  phase3.py — desire

  Nav tasks bootstrapped alongside desire tasks. The desire signature is
  goal_val appearing twice in the solution: once in gset (placing the goal in
  the world) and once in optimize/approach (driving the agent's behavior). Nav
  terminals use mode='full' (goal visible), desire terminals use mode='agent'
  (goal zeroed, so ECD must search over goal position and value). Stitch
  discovers fn_desire($ig, $gr, $gc, $gv) with $gv shared — that shared variable
   IS desire.

  ---
  phase4.py — physics vs. intention

  Tasks where an agent drifts in a straight line toward a goal at the opposite
  edge. Two explanations produce identical trajectories: step(1, dir) (physics,
  3 symbols) and optimize(neg_distance(2), 1) (intention, 4 symbols). The
  physics explanation is shallower, so ECD finds it first under any
  depth-penalizing prior. The point: simpler hypotheses win when evidence
  doesn't distinguish them. Stitch discovers fn_physics($ig, $dir) with
  direction as an explicit free parameter.

  ---
  phase5.py — goal algebra, sequential desire

  Two tiers of tasks: simple desire (agent approaches one goal) and sequential
  desire (agent approaches goal1 until consumed, then switches to goal2).
  root_type=fn so unfold is implicit. The key optimization: fn_want = 
  approach_from(1) is a core primitive so ECD doesn't need to assemble
  optimize(neg_distance(gv), 1) from scratch each occurrence, keeping sequential
   desire programs at 14 nodes (tractable). Stitch discovers the goal combinator
   fn_cond_desire($gv1, $gv2) = if_fn(exists($gv1), fn_want($gv1), 
  fn_want($gv2)).

  ---
  phase6.py (new) — multi-agent belief assignment

  Two agents (val 1 and 4) navigate simultaneously, each holding a different
  phantom wall belief. The synthesis target is an agent_belief function (int →
  grid): given an agent value, return its believed initial world. DSL contains
  assign(agent_val, believed_g, fallback) and no_belief(g). Solution: assign(1, 
  place_wall(ig_i, r1, c1), no_belief(place_wall(ig_i, r2, c2))). ig_i appears
  twice — both beliefs anchored in the same actual world. The evaluator rejects
  programs that assign identical beliefs to all agents, forcing assign to appear
   and the two grids to differ. Stitch discovers fn_two_beliefs($ig, $r1, $c1, 
  $r2, $c2) as the belief assignment abstraction.
